You are an expert human evaluator for summarization. Your job is to score a generated summary of a document using the SummEval dimensions: coherence, consistency, fluency, and relevance. IMPORTANT RULES - Use ONLY the information in the Source Document. Do not use outside knowledge. - If the summary makes a claim not supported by the Source Document, treat it as an inconsistency. - Be consistent and calibrated: use the full 1–5 scale when appropriate. - Think step-by-step internally, but DO NOT reveal hidden reasoning. Only output the final JSON. SCORING RUBRIC (1–5) Coherence: How logically organized and well-structured the summary is. - 5: Excellent flow; easy to follow; no awkward jumps. - 4: Mostly clear; minor ordering/transition issues. - 3: Some disjointedness; noticeable transition problems. - 2: Hard to follow; poor structure. - 1: Incoherent. Consistency (faithfulness to source): Whether all statements in the summary are supported by the source. - 5: All claims supported; no contradictions; no hallucinations. - 4: Mostly supported; at most 1 minor unsupported detail. - 3: Some unsupported or slightly incorrect statements, but main idea still matches. - 2: Multiple unsupported/incorrect statements or one major contradiction. - 1: Largely fabricated or contradicts the source. Fluency: Grammar, readability, and naturalness of the language. - 5: Perfect/near-perfect grammar and natural phrasing. - 4: Minor grammatical issues, but very readable. - 3: Noticeable errors or awkward phrasing, still understandable. - 2: Frequent errors that hurt readability. - 1: Very hard to read. Relevance: Whether the summary captures the most important information from the source (without major omissions or irrelevant content). - 5: Captures all key points concisely; no important omissions. - 4: Captures most key points; minor omission or small irrelevant detail. - 3: Covers some important points but misses others; or includes some irrelevant content. - 2: Misses many key points or focuses on unimportant details. - 1: Mostly irrelevant. G-EVAL-STYLE PROCEDURE (DO NOT OUTPUT THESE STEPS) 1) Identify 3–6 key points from the Source Document (mentally). 2) Check coverage: which key points are included/missed in the summary. 3) Check consistency: list any unsupported, exaggerated, or contradicting claims (mentally), and locate evidence spans in the source. 4) Judge coherence and fluency from the summary text itself. 5) Assign 1–5 scores for each dimension using the rubric. OUTPUT FORMAT (STRICT) Return ONLY a valid JSON object exactly in this schema: { "coherence": {"score": 1-5, "explanation": "..."}, "consistency": {"score": 1-5, "explanation": "...", "evidence": [{"summary_claim": "...", "source_quote": "..."}]}, "fluency": {"score": 1-5, "explanation": "..."}, "relevance": {"score": 1-5, "explanation": "..."} } EVIDENCE REQUIREMENT - In "consistency.evidence", include 0–3 items. - Each item must contain: - "summary_claim": a short claim from the summary (or paraphrase) that is potentially incorrect/unsupported OR especially important to verify. - "source_quote": an exact quote from the Source Document that supports or contradicts the claim. - If there are no issues and the summary is fully supported, return an empty list [] and say so in the explanation.
